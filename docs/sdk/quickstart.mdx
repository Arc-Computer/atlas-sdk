# Atlas Quickstart

The `atlas quickstart` command demonstrates Atlas learning capabilities with a series of security review tasks. It runs 3 progressively challenging tasks that show how Atlas learns and improves over time, displaying metrics and learning insights.

## Quick Start

```bash
# Install Atlas SDK
pip install arc-atlas

# Set your API key (or use --offline for testing)
export OPENAI_API_KEY=sk-...

# Run the quickstart demonstration
atlas quickstart
```

## What You'll See

The quickstart runs 3 security review tasks:

1. **Task 1 (Baseline):** Prompt injection vulnerability in a customer support bot
2. **Task 2 (Learning Applied):** Database access code review - shows learning transfer
3. **Task 3 (Generalization):** API authentication code review - demonstrates learning across security domains

## Command Options

### Basic Usage

```bash
# Run all 3 tasks (default)
atlas quickstart

# Run only 1 task
atlas quickstart --tasks 1

# Run 2 tasks
atlas quickstart --tasks 2
```

### Offline Mode

Test Atlas without making real API calls:

```bash
atlas quickstart --offline
```

This sets `ATLAS_OFFLINE_MODE=1` and uses mock responses. Learning telemetry will be limited, but you can see the command structure and output format.

### Custom Configuration

Use a different Atlas config file:

```bash
atlas quickstart --config configs/examples/openai_agent.yaml
```

### Storage Options

By default, quickstart checks if Postgres storage is available. If not found, it shows a warning and continues without persistence.

```bash
# Skip storage check entirely
atlas quickstart --skip-storage

# Storage is automatically checked if not skipped
# If unavailable, you'll see:
# âš ï¸  Postgres storage is not available.
#    Atlas quickstart can run without storage, but learning persistence will be disabled.
#    To enable storage, run: atlas init
```


## Troubleshooting

### Missing API Keys

If you see an error about missing `OPENAI_API_KEY`:

```bash
# Set the key
export OPENAI_API_KEY=sk-...

# Or use offline mode
atlas quickstart --offline
```

### Config File Not Found

If the default config is missing:

```bash
# Specify a custom config
atlas quickstart --config path/to/your/config.yaml
```

### Storage Issues

If Postgres is unavailable, quickstart will continue without persistence. To enable storage:

```bash
# Initialize Atlas storage
atlas init

# Then run quickstart again
atlas quickstart
```
